# Documentação Empat.IA

Este diretório mantém apenas a documentação operacional do projeto. O README principal continua sendo a porta de entrada pública; aqui ficam as referências para desenvolver, operar e decidir próximos passos.

## Documentos ativos

| Documento | Quando usar |
|---|---|
| [../README.md](../README.md) | Visão geral do produto, stack, setup rápido e roadmap resumido. |
| [roadmap/ROADMAP.md](roadmap/ROADMAP.md) | Roadmap vivo: prioridades concluídas, Prompt Control, RAG e voz. |
| [TECHNICAL.md](TECHNICAL.md) | Arquitetura, variáveis, API, banco, autenticação, prompts e deploy. |
| [FRONTEND.md](FRONTEND.md) | Rotas, componentes, estado e integração da Web UI e do Admin. |
| [CODEBASE_MAP.md](CODEBASE_MAP.md) | Onde encontrar os principais arquivos e responsabilidades no código. |
| [CONVENTIONS.md](CONVENTIONS.md) | Padrões de código, endpoints, componentes, banco, erros e commits. |
| [AGENTS.md](AGENTS.md) | Guia rápido para agentes de desenvolvimento neste repositório. |

## Regra simples

- Produto e narrativa ficam no README principal.
- Planos futuros ficam no roadmap.
- Detalhe técnico fica no TECHNICAL.
- Padrões de implementação ficam em CONVENTIONS.
- Mapas e navegação ficam em CODEBASE_MAP e AGENTS.
